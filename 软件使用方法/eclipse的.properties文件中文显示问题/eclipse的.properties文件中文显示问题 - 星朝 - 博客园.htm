eclipse的.properties文件中文显示问题

eclipse中打开含有汉字的properties文件，显示乱码；

保存前为:

user.name=对呀

保存后：

user.name=\u5BF9\u5440

properties文件编码为ISO8859-1，其实这不是乱码，只是将汉字转化为了ascii字符


我们仔细观察就会发现，字符都是\uxxxx这样的形式。 如果熟悉unicode 编码的话，我们应该知道\uxxxx形式是unicode的代码点，这里eclipse将汉字自动转化为ascii字符了。

所以，严格说来这不是乱码。

当然这个不是重点，重点是我们理解编码的原理；


这是由于properties的编码方式不正确导致，properties文件默认编码方式是iso8859-1,将其修改为utf-8后，就可以正确显示了。

右击属性文件---->属性------->resource----->text file encoding------>选择other utf-8





看到这里我们也许会想到，怎样将properties文件的默认编码方式改为utf-8呢？

window---->preferences---->general----->content types----->Text------>java properties file---->UTF-8---->update ------>ok

这里要注意，一定要点击update，否则不起作用。
